# Web UI для API календаря

Отдельный интерфейс для работы с API графиков (логин, загрузка активного периода, редактирование дней в календаре, отправка изменений обратно в API).

## Запуск

Самый простой способ — статический сервер из директории `web_ui`:

```bash
cd web_ui
python3 -m http.server 8080
```

Открыть в браузере: <http://localhost:8080>

## Что умеет

- Авторизация через `POST /auth/login`.
- Получение периода через `GET /periods/current`.
- Получение графика через `GET /schedules/me`.
- Локальное редактирование календаря через форму интерфейса (добавление/изменение/удаление дня без ручного JSON).
- Массовое добавление/изменение дней через диапазон дат в интерфейсе.
- Кнопка загрузки дефолтного тестового набора данных (14 дней) для быстрой проверки функций UI.
- Сохранение полного графика через `PUT /schedules/me`.
- Если активного периода нет, UI показывает причину (например, не задан `alliance` у пользователя) и включает локальный режим редактирования без отправки в API.

## Важно

API endpoint `PUT /schedules/me` в текущей реализации удаляет все существующие записи за период и создает новые на основе `payload.days`. Поэтому в интерфейсе отправляется весь текущий локальный набор дней.
